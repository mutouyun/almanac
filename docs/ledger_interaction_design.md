# Almanac Ledger 交互设计文档 v1.6

> 上游依据：`docs/ledger_requirements.md` (v1.10)
> 本文定义人机触点、页面布局与状态流转，作为后续数据建模与技术概要设计的输入。

## 0. 设计总览

### 0.1 访问模型
- **登录是硬门槛**：未登录无法看到任何账本数据、看板或入口。进站只呈现登录页。
- **登录即管理员**：任何用户登录后均拥有其账本内的完整权限（增删改查、分类维护、正则配置、CSV 导入），不做细粒度角色分级。
- **多用户数据隔离**：每个用户登录后仅能访问自己的账本，数据按 `user_id` 严格隔离。

### 0.2 三条核心链路
1. **记账链路（无人值守）**：iOS 快捷指令 → Webhook → 路由归类 → 入库。靠 `webhook_token` 鉴权，与登录态无关。
2. **登录闸门**：未登录一律拦截到登录页；登录成功进入管理应用。
3. **管理应用（登录后）**：看板页、账目明细页、分类树管理页、CSV 导入页。

### 0.3 全局导航流转
```mermaid
flowchart TD
    Start([进入网站]) --> Auth{已登录?}
    Auth -- 否 --> Login[登录页]
    Login -- 登录成功 --> Dashboard[看板页]
    Auth -- 是 --> Dashboard
    Dashboard <--> Entries[账目明细页]
    Dashboard <--> Categories[分类树管理页]
    Dashboard <--> Import[CSV 导入页]
    Entries <--> Categories
    Dashboard -- 登出 --> Login

    Webhook([iOS 快捷指令]) -. webhook_token .-> API[/Webhook 接口/]
    API -. 归类入库 .-> DB[(账本数据)]
    DB -. 展示 .-> Dashboard
```

## 1. 记账链路（Webhook 无人值守）

### 1.1 触点说明
手机端快捷指令是唯一的“写入触点”，用户在此不与网页交互，只发一个 HTTP 请求。

### 1.2 请求 / 响应契约
- **请求**：`POST /api/webhook/entry`
  - 鉴权：Header 携带 `X-Webhook-Token: <token>`（每用户独立）。
  - Body (JSON)：
    ```json
    { "date": "2026-07-05T14:30:00+08:00", "type": "瑞幸咖啡", "amount": "-19.9" }
    ```
  - 字段映射：`date → record_time`（带时区 ISO 8601，后端归一化为东八区墙钟，**主动舍秒**存为 `YYYY-MM-DD HH:mm`）；`type → raw_type`；`amount → amount_cents`（元→分）。
  - **金额强类型契约**：`amount` 以**字符串**传递（如 `"-19.9"`），避免客户端/服务端 JSON 序列化时的 IEEE 754 浮点精度丢失；后端用 `json.Number` 或字符串解析后交 decimal 处理，严禁 `float64` 接入参。
- **响应三态**：
  | 场景 | HTTP | 返回体要点 | 手机端可提示 |
  | :--- | :--- | :--- | :--- |
  | 命中分类 | 200 | 归类结果（分类名 + 完整路径） | “已记入：餐饮/饮品/咖啡” |
  | 落入待分类 | 200 | `status: unclassified`（接口实时计算返回，非存储字段；库中以 `category_id IS NULL` 隐式表达） | “已记账，待补分类” |
  | 鉴权/参数错误 | 401/400 | 错误原因 | “记账失败：token 无效” |

### 1.3 状态流转
```mermaid
flowchart LR
    A[收到请求] --> B{token 有效?}
    B -- 否 --> E1[401 拒绝]
    B -- 是 --> C{参数合法?}
    C -- 否 --> E2[400 拒绝]
    C -- 是 --> D[按 amount 定方向]
    D --> F[遍历全员正则\n深度优先匹配]
    F -- 命中 --> G[入库 + 溯源填父级]
    F -- 未命中 --> H[入库标记 待分类]
    G --> R1[200 返回归类结果]
    H --> R2[200 返回 待分类]
```

## 2. 登录闸门

### 2.1 线框：登录页
```
+--------------------------------------------------+
|                                                  |
|                Almanac 记账本                     |
|                                                  |
|        +----------------------------------+      |
|        |  用户名  [____________________]  |      |
|        |  密码    [____________________]  |      |
|        |            [   登  录   ]        |      |
|        +----------------------------------+      |
|                                                  |
|          未登录无法访问任何账本数据                |
+--------------------------------------------------+
```

### 2.2 交互反馈
- **用户操作**：输入用户名/密码 → 点「登录」。
- **系统反馈**：
  - 成功 → 建立会话（Session/JWT）→ 跳转看板页。
  - 失败 → 页内红字提示“用户名或密码错误”，不跳转。
- **会话保持**：已登录用户直接进站落到看板页，不再显示登录页。
- **登出**：应用内任意页可登出，清除会话回到登录页。

### 3.2 交互反馈
- **月份切换**：顶部时间选择器切月 → 卡片与图表刷新。
- **饱图下钻**：点击一级分类扇区 → 展开其子孙分类占比（逐层下钻，最多 5 层）。金额按递归 CTE 汇总。因账目可直挂非叶节点，下钻时额外渲染一个“本级直接”伪切片（= 本节点总额 − 各子节点总额之和），保证子项之和 = 父级总额。
- **待分类入口**：高亮提示条，点“去处理”跳转账目明细页的待分类筛选。
- **看板在登录后为可配置编辑页**：可调整展示时间范围、图表类型（后续能力）。

## 4. 账目明细页

### 4.1 线框
```
+-----------------------------------------------------------+
| Almanac  [看板] [明细] [分类] [导入] [设置]  木云 ▾ [登出] |
+-----------------------------------------------------------+
| 筛选: [月份 ▾] [分类 ▾] [□ 仅看待分类]  [批量操作] [+ 手动记一笔]|
+-----------------------------------------------------------+
| □  时间              原始描述     分类路径        金额     |
|-----------------------------------------------------------|
| □  07-05 14:30   瑞幸咖啡     餐饮>饮品>咖啡    -19.90    |
| □  07-05 09:00   工资         收入>薪资        +12000.00 |
| □  07-04 20:15   智波扫码     ⚠ 待分类 [关联] -35.00    |
+-----------------------------------------------------------+
```

**选择态线框（点「批量操作」后，行首 checkbox 生效 + 底部浮动操作栏）：**
```
+-----------------------------------------------------------+
| ☑ 全选   已选 2 笔 · 合计 -54.90            [退出选择]    |
|-----------------------------------------------------------|
| ☑  07-05 14:30   瑞幸咖啡     餐饮>饮品>咖啡    -19.90    |
| ☑  07-04 20:15   智波扫码     ⚠ 待分类 [关联] -35.00    |
| □  07-05 09:00   工资         收入>薪资        +12000.00 |
+-----------------------------------------------------------+
| 已选 2 笔 · 合计 -54.90   [改分类 ▾] [删除] [导出]         |
+-----------------------------------------------------------+
```

### 4.2 交互反馈
- **列表**：按 `record_time` 倒序（时光轴），支持按月份/分类筛选。
- **待分类行**：醒目标记，行内“关联”按钮 → 弹窗选择分类节点。关联后可选“同时为该原始描述生成一条正则规则”，实现“一次设置、终身免录”。
- **手动记一笔**：补充录入入口（餐饮弹窗填日期/分类/金额/备注），应对非 Webhook 的临时记账。

### 4.3 批量操作

面对一次导入或长期积累的大量账目，逐条改分类/删除效率太低。批量操作让用户先框选一批账目，再一次性改分类或删除。

**进入与退出选择态**
- 明细页顶部「批量操作」按钮进入选择态：每行左侧显出 checkbox，表头出现全选/反选 checkbox。
- 全选只对**当前筛选 + 当前页**的账目生效，避免误伤筛选范围外的数据。若列表分页且用户想跨页操作，需先调大筛选范围（如按月）再进选择态，不做隐式「选中全部符合条件」。
- 「退出选择」或完成一次操作后自动退回普通浏览态。

**悬浮操作栏（选择态下常驻）**
- 顶部条实时显示「已选 N 笔 · 合计金额」，合计由已选行金额累加，给用户一个确认锚点。
- 底部/顶部操作按钮：`改分类`、`删除`、`导出`（导出为后续能力）。未选中任何行时按钮置灰。

**批量改分类**
- 点「改分类」弹出分类选择器（复用明细行「关联」的分类树选择组件），选定目标节点后二次确认。
- 确认框展示目标分类的**完整路径**（如 `餐饮>饮品>咖啡`），并提示「方向将按新分类自动调整，可能影响本月收支合计」——因为方向由归类分类派生，跨方向改分类会翻转账目的收/支属性。
- 提交后 toast 反馈「成功改 N 笔」，失败/跳过的给原因（如某些 id 已被并发删除）。

**批量删除**
- 破坏性操作，必须弹确认框复述「将删除 N 笔，合计 ¥X，不可恢复」。
- 确认后 toast 反馈「已删除 N 笔」。

**结果反馈统一格式**：`成功 N 笔，跳过 M 笔`，跳过项可展开看原因。

### 4.4 批量操作状态流转
```mermaid
flowchart LR
    A[明细列表] --> B[点「批量操作」进入选择态]
    B --> C[勾选若干行 / 全选本页]
    C --> D{选择操作}
    D -- 改分类 --> E[选目标分类节点]
    E --> F[二次确认\n展示目标路径+方向提示]
    D -- 删除 --> G[二次确认\n复述笔数+合计+不可恢复]
    F --> H[提交批量接口\n单事务全或无]
    G --> H
    H --> I[toast: 成功 N / 跳过 M]
    I --> J[退出选择态\n列表刷新]
```

## 5. 分类树管理页

### 5.1 线框
```
+-----------------------------------------------------------+
| Almanac  [看板] [明细] [分类] [导入] [设置]  木云 ▾ [登出] |
+-----------------------------------------------------------+
| [支出树] [收入树]                        [+ 新建一级分类] |
+---------------------------+-------------------------------+
| ▾ 餐饮 (支出)   ≡拖拽     |  右侧：选中节点详情           |
|   ▾ 饮品        ≡       |  名称: [咖啡__________]      |
|     • 咖嘜         ≡       |  方向: 支出 (继承自父，锁定)  |
|     • 奶茶         ≡       |  层级: 3 / 5                 |
|   ▸ 正餐        ≡       |  --------------------------- |
| ▸ 交通 (支出)   ≡       |  任意节点均可配以下区域：       |
|                           |  正则: [瑞幸____________]    |
|                           |  (同层拖拽排序定匹配先后)     |
|                           |  [普通模式] / [高级正则模式]    |
+---------------------------+-------------------------------+
```

### 5.2 交互反馈
- **树形维护**：增/删/改节点；拖拽调整层级与父子关系（受 5 层上限约束）。
- **方向继承（视觉约束）**：子节点方向锁定为父节点方向，不可修改（创建后也不可改）；跨方向拖拽被禁止。
- **全员可配规则**：**任意层级节点**右侧都出现“正则”编辑区（不再限叶子）。
- **匹配优先级**：按**层级深优先**（子 > 父自动成立）；同一父下的兵弟由用户**上下拖拽**定先后（每一层均可拖）；拖拽序同时也是展示序。
- **普通 vs 高级模式**：默认普通模式（输入裸关键词，底层存裸词如 `瑞幸`，依托 Go regexp 非锥定特性天然实现**包含匹配**）；切换高级模式可手编正则（如 `^瑞幸$` 做精确匹配）。

### 5.3 新物种发现回路
```mermaid
flowchart LR
    A[待分类记录] --> B[明细页点“关联”]
    B --> C[选择分类节点]
    C --> D{生成规则?}
    D -- 是 --> E[写入该节点正则\n后续自动归类]
    D -- 否 --> F[仅归类本条]
    E --> G[该记录完成归类]
    F --> G
```

## 6. CSV 导入页

### 6.1 分步流程
```mermaid
flowchart LR
    A[上传 CSV] --> B[解析预览]
    B --> C[路由引擎试跑]
    C --> D{全部对齐?}
    D -- 是 --> F[确认入库]
    D -- 否 --> E[待确认清单\n手动校对]
    E --> F
    F --> G[导入完成报告]
```

### 6.2 交互反馈
- **上传**：选择飞书导出的 CSV；展示列映射预览（日期/类型/金额 列对应）。
- **试跑**：每行跑一遍路由引擎，展示归类结果。
- **待确认清单**：无法自动对齐的行单独列出，支持批量手动指定分类节点。
- **幂等去重**：基于（时间 + 原始描述 + 金额）查重，防止重复导入。
- **报告**：完成后展示成功/跳过/待分类条数统计。

## 7. 设置/账号安全页

### 7.1 线框
```
+-----------------------------------------------------------+
| Almanac  [看板] [明细] [分类] [导入] [设置]  木云 ▾ [登出] |
+-----------------------------------------------------------+
| 【账号安全】                                               |
|                                                           |
| Webhook Token（用于 iOS 快捷指令）                         |
| +-------------------------------------------------------+ |
| | alm_••••••••••••••••••••••••••••  [显示] [复制]      | |
| +-------------------------------------------------------+ |
| [重新生成]  ⚠️ 旧 token 立即失效，已配置的快捷指令需更新    |
|                                                           |
| 快捷指令配置示例                                            |
| +-------------------------------------------------------+ |
| | URL: https://你的域名/api/webhook/entry                | |
| | 方法: POST                                             | |
| | Header: X-Webhook-Token: <你的token>                  | |
| | Body (JSON):                                          | |
| |   {                                                   | |
| |     "date": "2026-07-05T14:30:00+08:00",            | |
| |     "type": "瑞幸咖啡",                               | |
| |     "amount": "-19.9"                                | |
| |   }                                                   | |
| +-------------------------------------------------------+ |
|                                                           |
| 【修改密码】                                               |
| 当前密码: [____________________]                          |
| 新密码:   [____________________]                          |
| 确认密码: [____________________]                          |
| [保存]                                                    |
+-----------------------------------------------------------+
```

### 7.2 交互反馈
- **Token 展示**：
  - 默认打码（`alm_••••••••`），点「显示」切换明文/打码。
  - 「复制」按钮一键复制完整 token 到剪贴板，成功后 toast 提示。
- **重新生成**：
  - 点「重新生成」弹窗二次确认（提示旧 token 失效、快捷指令需更新）。
  - 确认后调用 `/api/auth/webhook-token/regenerate`，成功后页面刷新展示新 token，并 toast 提示「新 token 已生成」。
- **配置示例**：
  - 可复制的代码块，方便用户粘贴到 iOS 快捷指令配置。
  - ⚠️ 生产环境需提示 HTTPS 必须性（防 token 明文泄露）。
- **修改密码**：
  - 校验当前密码正确性，新密码强度（≥8位，含字母数字），两次输入一致。
  - 成功后提示「密码已更新」，无需退出登录。

## 8. 页面及状态清单（汇总）
| 页面 | 登录要求 | 核心交互 |

| Webhook 接口 | token 鉴权 | 机器写入，无 UI |
